Observer Pattern

January 30, 2024

Observer Pattern

Problem

어느 한 고객이 새로운 제품의 재고확인을 위해 매일같이 스토어에 방문한다고 가정해 봅시다.

매일같이 방문하는 것은 너무 비효율적인 것 같습니다.

언제 재고가 들어올지 정확하게 모르고, 그렇다고 너무 자주가기에는 시간이 없고,,

그럼 우리는 사장님한테 이렇게 말할거 같아요, 재고가 들어오면 (상태변화) 이 번호로 연락을 달라고 (구독)

이게 옵저버 패턴입니다.

Solution

그런데 사장님 입장에서는 고객이 이 사람만 있는게 아니기 때문에 어떤 고객이 재입고 정보를 알고 싶은지 알아야합니다.

옵저버 패턴에서는 구독이라는 방식을 통해 문제를 해결합니다.

어떠한 상태값의 변동을 원하는 객체는 옵저버 구독을 통해 변동에 대한 값을 받을 수 있습니다.

객체에 직접 물어보는 방식이 아닌 객체가 알아서 알려주는 방식이라고 보시면 됩니다.

이렇게 되면 구독, 구독해지를 통해 쉽게 연결을 관리할 수 있고 두 객체간의 결합도를 매우 낮출 수 있습니다. (느슨한 결합)

Code

NewsPublisher.java 공급자, 사장님 구현코드

import java.util.ArrayList;  
  
public class NewsPublisher implements Publisher{  
	private ArrayList<Observer> observers;  
	private String title;  
	private String news;  
  
	public NewsPublisher() {  
		observers = new ArrayList<>();  
		title = null;  
		news = null;  
	}  
  
	@Override  
	public void registerObserver(Observer observer) {  
	observers.add(observer);  
	}
	  
	@Override  
	public void removeObserver(Observer observer) {  
	observers.remove(observer);  
	}
	  
	@Override  
	public void notifyObservers() {  
		for(Observer observer : observers) {  
		observer.update(title, news);  
		}  
	}  
  
	public void setNews(String title, String news) {  
	this.title = title;  
	this.news = news;  
	notifyObservers();  
	}  
}

registerObserver() 로 구독을 진행하고
removeObserver() 로 구독을 해제할 수 있습니다.
notifyObservers() 로 구독한 사람에게만 news를 전송합니다.

NewsSubscriber.java 구독자, 고객 구현코드

public class NewsSubscriber implements Observer{  
	private String observerName;  
	private String news;  
	private Publisher publisher;  
	  
	public NewsSubscriber(String subscriber, Publisher publisher) {  
	this.observerName = subscriber;  
	this.publisher = publisher;  
	publisher.registerObserver(this);  
	}  
	  
	@Override  
	public void update(String title, String news) {  
	this.news = title + "!!! " + news;  
	display();  
	}  
	  
	private void display() {  
	System.out.println("=== " + observerName + " 수신 내용 ===\n" + news + "\n");  
	}  
}

생성자에서 어떤 뉴스를 구독할 지 주입하게 됩니다.
구독자의 setNews() 호출 시 변경된 상태에 대해 데이터를 받아 출력합니다.

Main.java 실행코드

public class Main {  
  
	public static void main(String[] args) {  
	NewsPublisher newsPublisher = new NewsPublisher();  
	  
	NewsSubscriber newsSubscriber1 = new NewsSubscriber("구독자 1", newsPublisher);  
	NewsSubscriber newsSubscriber2 = new NewsSubscriber("구독자 2", newsPublisher);  
	  
	newsPublisher.setNews("날씨", "내일 아침 '영하 8도'..모레 낮부터 누그러져");  
	newsPublisher.setNews("금융", "\"5년 만기 부담\"…청년 현실 외면하는 청년금융상품");  
	  
	newsPublisher.removeObserver(newsSubscriber1);  
	newsPublisher.setNews("경제", "고금리 못 버텨…빚 못갚아 경매 넘어간 부동산 급증");  
	}  
}

=== 구독자 2 수신 내용 ===
날씨!!! 내일 아침 '영하 8도'..모레 낮부터 누그러져

=== 구독자 1 수신 내용 ===
금융!!! "5년 만기 부담"…청년 현실 외면하는 청년금융상품

=== 구독자 2 수신 내용 ===
금융!!! "5년 만기 부담"…청년 현실 외면하는 청년금융상품

=== 구독자 2 수신 내용 ===
경제!!! 고금리 못 버텨…빚 못갚아 경매 넘어간 부동산 급증

두명의 구독자를 생성 후 NewsPublisher의 상태가 변경될 때마다 구독자는 데이터를 수신받습니다.
newsPublisher.removeObserver(newsSubscriber1) 이후에는 구독자 2만이 수신받게 됩니다.
코드상에서 NewsSubscriber 는 생성시 구독정보만 주입할 뿐 어떠한 행위를 하지 않았습니다.

Applicability

옵서버 패턴은 한 객체의 상태가 변경되어 다른 객체들을 변경해야 할 필요성이 생겼을 때, 그리고 실제 객체 집합들을 미리 알 수 없거나 이러한 집합들이 동적으로 변경될 때 사용하기에 적합한 패턴입니다.
